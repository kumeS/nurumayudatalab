クラウドフレア側の実装を変更しました。

io.netのAPI endpointが頻繁に変わるので、利用可能モデルを取得する機能を追加しました。

index.htmlにアクセス時、ページ更新時に利用可能モデルを取得してモデル一覧として選択できるようにします。

以下がcurlでの実行例ですので、参考にしてください。

https://nurumayu-worker.skume-bioinfo.workers.dev/modelsを実行して、model一覧を取得してそのモデル名を抽出してリストを作成します。また、それを随時利用します。

  # 1. 利用可能モデルを確認
  curl https://nurumayu-worker.skume-bioinfo.workers.dev/models | jq '.data[].id'

 # 2. コスパの良いモデルでチャット
 curl -X POST https://nurumayu-worker.skume-bioinfo.workers.dev \
    -H "Content-Type: application/json" \
    -d '{
      "model": "openai/gpt-oss-120b",
      "messages": [
        {
          "role": "user",
          "content": "JavaScript で配列をシャッフルする効率的な方法を教えて"
        }
      ],
      "max_completion_tokens": 300,
      "stream": false
    }'


モデル説明ページに、gpt-oss-120Bとgpt-oss-20Bの説明をNEWとして追加して。
